### 3. ノイズモデルのサポート

MQTのノイズツールを使用して実装:

```python
from mqt.qudits.simulation.noise_tools import Noise, NoiseModel

# ノイズモデルの作成
noise = Noise(
    probability_depolarizing=0.05,  # 5%のエラー率
    probability_dephasing=0.03       # 3%のエラー率
)

noise_model = NoiseModel()
noise_model.add_all_qudit_quantum_error(
    noise, 
    ["x", "h", "rz", "r", "custom_one"]  # すべてのゲートタイプに適用
)

# シミュレータで使用
sim = MQTShotSimulator(trotter_order=2, noise_model=noise_model)
```

**ノイズタイプ:**
- **脱分極化ノイズ (Depolarizing)**: 量子状態をランダムに混合状態に置き換える
- **デフェージングノイズ (Dephasing)**: ランダムに位相エラーを適用
- **設定可能**: すべてのゲートまたは特定のゲートタイプに適用可能

## 重要な設計決定

### 1. ヒューリスティックやフォールバックを使用しない

**決定:** MQT Quditsバックエンドの機能のみを使用
**理由:** 正確性と保守性を確保
**実装:** MQTのstochasticシミュレーションAPIの直接使用

### 2. ノイズなしシミュレーションのための最小ノイズ

**課題:** MQTはノイズモデルが存在する場合のみショットシミュレーションを実行
**解決策:** 無視できるエラー率（≈10⁻¹²）の「最小ノイズ」モデルを作成
**結果:** 結果に影響を与えずにショットシミュレーションを実現

### 3. 統計誤差の推定

**決定:** すべてのショットベース測定に対して標準誤差を計算・報告
**実装:** 
- 期待値: σ = sqrt((⟨O²⟩ - ⟨O⟩²) / N_shots)
- 占有確率: σ = sqrt(P(1-P) / N_shots)
**利点:** ユーザーが測定の信頼性を評価可能

### 4. Zスコア検証

**決定:** 統計的一貫性チェックのためのZスコアメトリクスを含める
**実装:** Z = (measurement_shot - measurement_exact) / σ_shot
**利点:** ショットシミュレーションが統計的に正しく動作することを検証

## 検証結果

### テスト1: ノイズなしショットシミュレーション
- **ハミルトニアン:** Zeeman効果 H = -ω₀Jz
- **初期状態:** |m=+1⟩
- **ショット数:** 500
- **結果:** 
  - 厳密解と完全に一致（fidelity = 1.0）
  - すべての測定が期待される基底状態
  - Zスコア < 3（統計的に一貫）

### テスト2: 比較メソッド
- **ハミルトニアン:** Zeeman効果
- **初期状態:** (|m=+1⟩ + |m=0⟩)/√2
- **ショット数:** 500
- **結果:**
  - Statevector fidelity: > 0.99999999
  - ショット fidelity: > 0.99999999
  - 最大Zスコア: < 1.0
  - すべての方法が統計誤差内で一致

### テスト3: ノイズありシミュレーション
- **ハミルトニアン:** 横磁場 H = -ω_z Jz - ω_x Jx
- **ノイズ:** 5%脱分極化 + 3%デフェージング
- **ショット数:** 1000
- **結果:**
  - 期待値にノイズ効果が可視化
  - 時間とともに誤差が増加（予想通り）
  - 統計誤差が適切に計算
  - ノイズありの結果が厳密解から逸脱

## パフォーマンス特性

### 計算コスト
- **時間点あたり:** O(shots × n_gates × d³) ここでd=3（qutrit）
- **典型的:** 1000ショットで時間点あたり約0.5-1秒
- **スケーラブル:** 高速反復のためショット数を削減可能

### メモリ使用量
- **ストレージ:** O(shots × n_times × 3)
- **典型的:** 1000ショット × 100時間点で < 10 MB
